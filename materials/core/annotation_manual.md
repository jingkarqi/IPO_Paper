# 文本标注规范（适用于人工标注与 LLM 提示词）

## 一、标注目标

标注围绕三个主维度展开：

1. **提及强度**：由脚本自动计算，人工只需校验是否存在异常高频但明显无效的文本；
2. **场景具体性**：识别企业是否把 AI / 数字化说到了具体业务场景、流程或技术对象；
3. **表述审慎性**：识别企业表述是偏事实陈述、边界清晰，还是偏宣传式、承诺式、概念式。

同时增加两个辅助字段：

- **包装风险**：帮助构造“包装性”对照指标；
- **证据支撑**：不进入基准真实性指数，但可用于稳健性或辅助解释。

## 二、标注单元

### 段落级标注

先对“相关段落”逐段打分，字段如下：

- `relevance`：是否与企业自身 AI / 数字化披露直接相关（0/1）
- `scene_specificity`：场景具体性（0-3）
- `prudence`：表述审慎性（0-3）
- `packaging_risk`：包装风险（0-3）
- `evidence_support`：证据支撑（0-2）
- `notes`：一句话说明理由

### 企业级聚合

每家企业最终保留：

- 相关段落的平均分；
- 或“最具代表性段落”的复核得分；
- 建议同时保留 `max` 与 `mean` 两种聚合值，便于稳健性检验。

## 三、评分规则

## 1. 场景具体性 `scene_specificity`（0-3）

### 0 分：纯口号 / 纯概念

特征：
- 只说“公司积极推进数字化转型”；
- 只说“AI赋能公司发展”；
- 没有出现业务流程、技术对象、应用环节。

### 1 分：有方向，但仍较抽象

特征：
- 提到“应用于生产、营销、供应链、风控等领域”；
- 但没有说明具体任务、系统、对象或输出。

### 2 分：有明确业务环节或技术对象

特征：
- 说明了具体环节，如“智能质检”“设备运维预测”“仓储调度优化”；
- 或说明技术对象，如“机器学习模型用于缺陷识别”；
- 但尚未充分交代实施深度、结果或组织嵌入。

### 3 分：场景清晰且可落地

特征：
- 同时出现“具体业务环节 + 技术对象/系统 + 目标或输出”；
- 例如“公司在锂电池质检环节部署机器视觉模型，用于识别极片瑕疵并降低误检率”。

## 2. 表述审慎性 `prudence`（0-3）

### 0 分：明显宣传式或承诺式

特征：
- 大量出现“行业领先、全面重构、革命性、引领未来、显著提升”等词；
- 几乎没有边界、条件、阶段或不确定性说明。

### 1 分：宣传与事实混合

特征：
- 有一定事实描述，但整体仍偏“讲故事”；
- 对实施阶段、适用范围、限制条件交代不足。

### 2 分：基本客观、偏事实陈述

特征：
- 更像业务说明或技术说明；
- 有阶段性、范围、应用对象等描述；
- 宣传色彩较弱。

### 3 分：高度审慎、边界清楚

特征：
- 明确写明“正在建设/试点/探索”“存在不确定性”“尚未形成大规模应用”等；
- 同时说明了适用范围或约束条件。

## 3. 包装风险 `packaging_risk`（0-3）

### 0 分：几乎无包装风险

- 内容具体；
- 事实与边界交代充分；
- 与企业自身业务高度匹配。

### 1 分：轻度包装

- 有少量口号词，但主要内容仍较扎实。

### 2 分：中度包装

- 口号与概念词较多；
- 场景信息有限；
- 可能更像资本市场叙事而非经营实施说明。

### 3 分：高度包装

- 反复强调 AI/数字化；
- 但几乎没有可落地场景、证据或边界；
- 典型“说得很多，做得不清”。

## 4. 证据支撑 `evidence_support`（0-2）

### 0 分：无证据

- 没有研发、人员、系统、专利、项目等支撑。

### 1 分：弱证据

- 有一般性投入说明，如“持续研发”“组建团队”，但缺少客观量化信息。

### 2 分：强证据

- 有专利、系统上线、技术团队、募投项目、研发投入、客户落地、项目名称等较硬证据。

## 四、推荐的人工-LLM协同流程

1. 用脚本抽取相关段落；
2. 让 LLM 对每段先给出 JSON 格式初评；
3. 人工复核 80-150 家企业；
4. 对分歧较大的企业进行二次复核；
5. 形成企业级最终标签。

## 五、冲突处理规则

- 如果 LLM 与人工只差 1 分，可由人工直接裁定；
- 如果差 2 分及以上，必须回看原文；
- 若一个企业的招股书与回复函结论不一致，优先参考**回复函**对“真实性”的修正信息，但保留两源评分，便于后续做“问询修正效应”的扩展分析。

## 六、推荐的最终企业级字段

- `firm_id`
- `scene_specificity_mean`
- `prudence_mean`
- `packaging_risk_mean`
- `evidence_support_mean`
- `reviewer`
- `review_date`
- `confidence_level`
- `final_notes`
