# 文本标注规范

本规范服务于“LLM 初步评分 + 人工复核”的轻量标注流程。单位以“企业-文本片段”为主，最终汇总到“企业”层面。

## 一、标注对象

- 招股说明书中与 AI / 数字化相关的段落。
- 问询回复中与 AI / 数字化相关的回复段落。

每个片段应记录：

- `firm_id`
- `document_type`
- `passage_id`
- 原文片段
- LLM 初始判断
- 人工复核分值
- 备注

## 二、核心维度与评分标准

### 1. 提及强度

定义：企业对 AI / 数字化相关内容的披露频率、篇幅和可见度。

建议评分：

- `0分`：无实质相关内容。
- `1分`：偶发提及，仅 1-2 次关键词命中。
- `2分`：有若干相关段落，但篇幅有限。
- `3分`：相关内容较集中，覆盖多个段落或章节。
- `4分`：高频、多场景、多章节披露。

判定提示：

- 优先结合关键词次数、相关段落数和相关文本占全文比例。
- 提及强度高不等于真实性高，它只代表“说得多”。

### 2. 场景具体性

定义：企业是否把 AI / 数字化表述落实到具体业务环节、流程节点、系统模块或应用结果。

建议评分：

- `0分`：纯口号，无任何场景。
- `1分`：只有抽象战略表述，如“推进数字化转型”。
- `2分`：提到业务领域，但没有说明具体流程或机制。
- `3分`：说明了明确业务场景，如生产、营销、客服、供应链等。
- `4分`：不仅有场景，还给出了流程、系统、投入、数据基础或应用效果。

高分特征：

- 出现“在哪个部门/流程应用”“如何应用”“带来什么变化”。
- 出现系统名称、平台名称、技术团队、项目名称、实施阶段。

### 3. 表述审慎性

定义：企业表述是否克制、边界清晰、避免夸大承诺。

建议评分：

- `0分`：明显夸大、承诺化、宏大叙事。
- `1分`：存在较多包装性表达，缺乏边界说明。
- `2分`：既有业务信息，也混杂较强宣传语气。
- `3分`：整体较为审慎，偶有宣传性措辞。
- `4分`：明确交代适用范围、实施阶段、限制条件和不确定性。

高分特征：

- 明确“目前”“拟”“部分”“处于建设阶段”等边界。
- 区分“已实现”和“计划推进”。
- 少用“全面赋能”“行业领先”“颠覆式”等宣传性词汇。

### 4. 包装性补充评分

定义：文本是否存在明显概念化修辞、空泛战略口号和过度前瞻承诺。

建议评分：

- `0分`：几乎无包装性语言。
- `1分`：轻微包装。
- `2分`：中等包装。
- `3分`：包装性明显。
- `4分`：强包装，远多于具体事实。

说明：

包装性不是独立的“坏企业”标签，而是对叙事方式的度量。最终可作为真实性指数的反向维度，或单独构建包装性指数。

## 三、辅助判断标签

除主评分外，建议增加以下二元标签：

- `has_concrete_scenario`: 是否出现明确业务场景。
- `has_objective_evidence`: 是否出现研发、专利、团队、系统建设等客观证据。
- `regulator_questioned`: 问询中是否被监管关注真实性、先进性、必要性或披露充分性。
- `future_promise_heavy`: 是否以未来规划和愿景性措辞为主。

## 四、LLM 提示词使用原则

1. 先让模型摘要相关段落，再要求逐维评分。
2. 要求模型给出“引用依据句”而不是只给分数。
3. 明确要求模型区分“已经实施”与“计划实施”。
4. 最终正式分值以人工复核为准。

## 五、人工复核规则

1. 每家企业至少抽查 2 个关键片段。
2. 对于 `场景具体性=0 或 4`、`包装性=0 或 4` 的极端分值，优先复核。
3. LLM 与人工分值相差 2 分及以上时，必须写一句简短原因。

## 六、企业层面汇总建议

- 提及强度：可直接取全文统计值并标准化。
- 场景具体性：取企业相关片段平均分，或取最高分与平均分的加权值。
- 表述审慎性：取人工复核片段均值。
- 包装性：取片段均值，或与包装关键词比例结合。
